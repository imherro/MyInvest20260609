# 文档变更说明

## 本次变更

根据最新讨论，文档做了两类调整：

1. 将原来“P0a 未连续 5 个交易日稳定前，不得推进 P0b / P0c”的硬等待，改为“连续开发、分层启用”。
2. 新增架构设计、连续开发计划和 Codex 开发任务清单，用于指导 Codex 按顺序实现。

## 当前推荐开发方式

```text
需求文档定边界
架构文档定主结构
开发计划定顺序
Codex 任务清单定每次交付
Codex 在边界内自行实现代码细节
```

## 核心结论

现在可以开始开发，但不建议只把 FEATURES.md / REQUIREMENTS.md 丢给 Codex 自由发挥。

更稳方式：

1. 先让 Codex 阅读全部文档。
2. 让 Codex 输出自己的实现计划，但不得改变边界。
3. 你确认后，每次只给 Codex 一个 M 阶段任务。
4. 每个阶段必须有测试、feature flag 和安全扫描。
5. 后续功能可以连续开发，但未通过门禁前不得接入日常生产流水线。
