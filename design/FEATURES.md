# 功能清单

## 目标

建设一个本地大模型辅助的投资研究与风控系统，保存证据、结构化研究快照、决策记录和影子账户历史，便于复盘。

## 核心功能

1. 数据源
   - Tushare：A 股结构化主源。
   - QMT：盘中价格和真实持仓只读导入。
   - BaoStock：A 股交叉验证。
   - FRED / yfinance / 网络：宏观、海外和补充证据。

2. 标的池
   - 由主线 ETF、主线龙头、防御 ETF、现金 ETF、真实持仓和观察标的组成。
   - 真实持仓从 QMT 导入后自动进入标的池。
   - 标的池每日盘后自动估值和更新状态。

3. 研究历史
   - 市场、主题、ETF、个股、组合和估值研究都保存带时间戳的 Markdown 报告和 JSON 快照。
   - 关键数值结论写入 SQLite，用于曲线、筛选和历史对比。

4. 每日自动研究
   - Codex 每个交易日自动执行仓位、主线、龙头、ETF、个股估值和组合研究。
   - 自动保留原始数据、结构化快照、初稿报告和运行日志。
   - 初稿不直接作为最终报告。

5. ChatGPT 审核定稿
   - Codex 生成审核包，提交给网页版 ChatGPT 复核。
   - ChatGPT 检查数据缺口、逻辑冲突、风险遗漏和结论强度。
   - 通过审核后生成最终研究报告，并写入历史库和决策日志。

6. 市场仓位
   - 每个交易日研究一次。
   - 保存建议权益仓位区间、分层配置、市场评分、拥挤度惩罚、数据日期和生成时间。
   - 将建议仓位区间与主要指数走势对比展示。

7. 标的估值
   - 记录每次股票或 ETF 研究。
   - 保存合理价格区间、评级、置信度、估值方法、当前价格、失效条件和下次复查日期。
   - 展示合理价格区间的历史移动。

8. 组合分析
   - 只使用比例、百分点、目标区间和风险桶。
   - 不暴露总资产、持仓金额、股数、收益金额或真实订单细节。
   - 未完成研究的标的先进入 ResearchFirst，再讨论操作建议。

9. 持仓对比
   - 从 QMT 只读导入真实持仓快照。
   - 支持真实持仓、影子账户和主要指数的收益曲线、回撤、行业/主题暴露对比。
   - 对外展示使用比例和指数化曲线，不展示真实金额和股数。

10. 影子账户
   - 纸面模拟账户，每个交易日按理想仓位策略自动模拟交易。
   - 持仓只能由 AI 在标的池中选择，不能买入池外标的。
   - 生成模拟交易记录、持仓历史、资金曲线、回撤、换手率和基准对比。
   - 仅用于对比模型策略和真实账户，绝不发送真实订单。

11. 决策日志
   - 追加式记录结论变化。
   - 保存旧结论、新结论、证据、原因、影响、失效条件和下次复查日期。

12. Web 工作台
   - 只读展示当前状态、历史曲线、研究缺口、影子账户表现和复查任务。
   - 默认不提供真实交易入口。

## 首期范围

1. SQLite 表结构：标的池、研究记录、市场仓位快照、估值快照、组合快照、决策记录、影子账户记录。
2. 导入器：把 Markdown + JSON 研究结果写入 SQLite。
3. QMT 真实持仓只读导入和标的池自动更新。
4. 每日研究任务编排、盘后估值和 ChatGPT 审核包。
5. 每日仓位历史曲线、真实持仓、影子账户和指数基准对比。
6. 影子账户每日模拟交易引擎和资金曲线。

## 自动化需求

自动化由 Codex 自动化能力调度执行；表中的 AI 默认指 Codex 负责生成、分析、解释和审核前初稿。

| 任务 | 实现方式 | 频次 | 说明 |
|---|---|---|---|
| 交易日判断 | 脚本 | 每日盘前 | 使用交易日历决定是否运行。 |
| 数据采集 | 脚本 | 盘后每日，盘中按需 | Tushare 拉取盘后数据，QMT 只读确认盘中价格和真实持仓。 |
| 数据校验 | 脚本 | 每次采集后 | 检查缺失、过期、冲突和来源异常。 |
| 标的池更新 | 脚本 + AI | 盘后每日 | 脚本导入真实持仓，AI 维护主线 ETF、龙头、防御和观察标的。 |
| 盘后估值 | 脚本 + AI | 盘后每日 | 脚本计算基础指标，AI 给出估值解释、风险和区间。 |
| 市场仓位研究 | 脚本 + AI | 每个交易日 | 脚本生成市场事实包，AI 输出仓位区间和理由。 |
| 主线和龙头研究 | AI | 每个交易日 | 基于事实包判断主线强弱、龙头状态和失效条件。 |
| ETF 和个股研究 | AI | 每日按队列 | 标的池和 ResearchFirst 队列优先。 |
| 组合分析 | 脚本 + AI | 每个交易日 | 脚本计算比例和暴露，AI 输出解释和调整建议。 |
| ChatGPT 审核包 | 脚本 | 每个交易日 | 打包初稿、JSON、数据缺口、冲突项和待审核问题。 |
| 最终报告生成 | AI + 人工确认 | 审核后 | 网页版 ChatGPT 审核通过后定稿。 |
| 影子账户交易 | 脚本 | 每个交易日审核后 | 只在标的池内按最终策略生成纸面模拟交易。 |
| 历史曲线更新 | 脚本 | 每次入库后 | 更新仓位、估值、真实持仓、影子账户和指数对比。 |
| 异常提醒 | 脚本 | 每次任务后 | 提醒数据源失败、QMT 未登录、估值大幅变化或偏离目标。 |
| 安全检查 | 脚本 | 每次发布前 | 检查密钥、金额、股数、真实订单和边界违规。 |

详细验收规则、状态流和失败处理见 `design/REQUIREMENTS.md`。
